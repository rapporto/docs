Интеграция по HTTP
==================

Введение
---------

Компания (далее Сервис-провайдер) предоставляет организациям услуги по проведению рассылок с помощью собственной технологической платформы.
HTTP-интеграция позволяет Партнёрам отправлять сообщения пользователям и получать ответные сообщения, а также следить за статусами отправленных сообщений.

Для того, чтобы воспользоваться пакетом предлагаемых услуг Партнёру необходимо подключиться к Сервис-провайдеру по данному протоколу
взаимодействия.

В данном документе представлена следующая информация по HTTP-интеграции:

* данные, необходимые для настройки сервиса;

* параметры сообщений различных типов и примеры их отправки/получения;

* настройки доотправки сообщений различных типов;

* информация о методах получения статусов сообщений.

Описание
--------

Для отправки сообщения абоненту Партнёр обращается по HTTP-протоколу к определенному URL Сервис-провайдера и передает номер абонента, текст сообщения и прочие параметры. Сервис-провайдер принимает запрос, обрабатывает его, отправляет сообщение абоненту и возвращает Партнёру идентификатор сообщения. Для получения статуса доставки используется отдельный HTTP-запрос (этот функционал подробно описан в разделе «Сервис получения статусов доставки сообщений»).

Поддерживаются следующие типы сообщений:

* SMS;

* Viber;

* VK;

* WhatsApp;

* PUSH;

* FlashingCall (Voice Code).
